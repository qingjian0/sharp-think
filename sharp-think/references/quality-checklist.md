# 质量检查清单

> 在 L2 及以上任务输出前，根据输出类型选择对应的检查清单进行自检。

## 通用检查（所有输出类型）

- [ ] **方向正确**：回答了用户真正问的问题
- [ ] **可执行性**：有明确的下一步动作
- [ ] **边界考虑**：考虑了边界情况和常见错误
- [ ] **结构清晰**：输出有清晰的层次和逻辑关系
- [ ] **语言简洁**：没有冗余信息，每句话都有价值
- [ ] **术语一致**：全文使用统一的术语，没有前后不一致

---

## 文档类输出（报告/方案/周报/总结）

- [ ] **有明确标题**：读者一眼就知道文档主题
- [ ] **有执行摘要**：开头有 2-3 句话的总结，忙碌的读者只看摘要也能获取核心信息
- [ ] **逻辑连贯**：段落之间有清晰的逻辑关系（因果/递进/并列）
- [ ] **数据支撑**：关键论断有数据或具体案例支撑
- [ ] **格式规范**：标题层级正确、列表使用恰当、重点内容有加粗/高亮
- [ ] **结尾有行动项**：读者看完知道下一步该做什么

---

## 分析类输出（数据分析/问题诊断/趋势判断）

- [ ] **数据来源标注**：数据从哪来，是否可靠
- [ ] **分析方法说明**：用了什么分析方法，为什么选这个方法
- [ ] **结论有依据**：每个结论都能追溯到具体的数据/事实
- [ ] **局限性说明**：分析有哪些局限，结论在什么条件下成立
- [ ] **区分事实与推断**：明确标注哪些是事实、哪些是推断/假设
- [ ] **可复现性**：别人用同样的数据和方法能否得到类似结论

---

## 方案类输出（设计方案/解决方案/技术选型）

- [ ] **问题定义清晰**：方案要解决什么问题，一句话说清
- [ ] **多方案对比**：至少考虑了 2 个可选方案
- [ ] **选择理由充分**：推荐方案的选择理由具体且有说服力
- [ ] **实施路径明确**：有分步骤的执行计划，每步有可验证的产出
- [ ] **资源需求明确**：需要什么人/时间/工具/预算
- [ ] **风险评估到位**：主要风险已识别并有应对措施
- [ ] **成功标准可量化**：如何衡量方案是否有效

---

## 创意类输出（文案/内容创作/头脑风暴）

- [ ] **目标受众明确**：为谁而写/做
- [ ] **核心信息清晰**：最想让受众记住的一个点是什么
- [ ] **调性一致**：全文风格统一（正式/轻松/专业/亲切）
- [ ] **有吸引力的开头**：前 3 秒/前 3 行能抓住注意力
- [ ] **有行动号召**：受众看完应该做什么

---

## 代码类输出（脚本/工具/自动化）

- [ ] **可运行**：代码可以直接运行，不缺少依赖
- [ ] **有注释**：关键逻辑有注释说明
- [ ] **有错误处理**：预期可能出错的地方有 try-catch 或校验
- [ ] **有使用说明**：如何安装、运行、配置
- [ ] **输入输出明确**：输入什么、输出什么、格式是什么
