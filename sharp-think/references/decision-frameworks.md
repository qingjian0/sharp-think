# 决策框架速查

> 在 L3 深度分析任务中，当需要多方案对比或复杂决策时，选择合适的分析框架。

## 框架选择指南

| 场景 | 推荐框架 | 适用条件 |
|------|---------|---------|
| 战略方向选择 | SWOT 分析 | 需要从内外部多维度评估 |
| 多方案量化对比 | 决策矩阵 | 有 3+ 个方案，且可量化评估维度 |
| 投入产出评估 | 成本收益分析 | 需要衡量经济/时间投入与回报 |
| 优先级排序 | 优先级矩阵（影响-努力） | 有多个待办事项需要排序 |
| 风险评估 | 风险矩阵 | 需要识别和评估潜在风险 |
| 根因分析 | 5-Why 分析 | 需要找到问题的根本原因 |
| 全局视角 | 第一性原理 | 需要跳出既有框架重新思考 |

---

## 1. SWOT 分析

```
┌─────────────────┬─────────────────┐
│   优势 (S)       │   劣势 (W)       │
│  内部有利因素     │  内部不利因素     │
├─────────────────┼─────────────────┤
│   机会 (O)       │   威胁 (T)       │
│  外部有利因素     │  外部不利因素     │
└─────────────────┴─────────────────┘
```

**使用方法**：
1. 列出 3-5 个优势（S）和劣势（W）
2. 列出 3-5 个机会（O）和威胁（T）
3. 交叉分析：SO 策略（发挥优势抓住机会）、WO 策略（克服劣势利用机会）、ST 策略（利用优势应对威胁）、WT 策略（减少劣势规避威胁）

## 2. 决策矩阵

```
| 评估维度    | 权重 | 方案 A | 方案 B | 方案 C |
|-----------|------|--------|--------|--------|
| 维度 1     | W1   | 评分   | 评分   | 评分   |
| 维度 2     | W2   | 评分   | 评分   | 评分   |
| 维度 3     | W3   | 评分   | 评分   | 评分   |
|-----------|------|--------|--------|--------|
| 加权总分   | 100% | Σ      | Σ      | Σ      |
```

**使用方法**：
1. 确定 3-5 个关键评估维度
2. 为每个维度分配权重（总和 = 100%）
3. 对每个方案在每个维度上打分（1-5 分或 1-10 分）
4. 计算加权总分，选择最高分方案

**注意**：权重分配本身可能带有偏见，建议用「如果去掉这个维度，结论会变吗？」来检验。

## 3. 成本收益分析

```
|            | 方案 A | 方案 B |
|-----------|--------|--------|
| 直接成本   |        |        |
| 间接成本   |        |        |
| 机会成本   |        |        |
|-----------|--------|--------|
| 直接收益   |        |        |
| 间接收益   |        |        |
|-----------|--------|--------|
| 净收益     |        |        |
| ROI       |        |        |
| 回收周期   |        |        |
```

**使用方法**：
1. 列出所有成本类型（直接/间接/机会）
2. 列出所有收益类型（直接/间接）
3. 量化（尽量用数字，无法量化则标注为"高/中/低"）
4. 计算净收益和 ROI

---

## 4. 优先级矩阵（影响-努力）

```
        高影响
          │
    ┌─────┼─────┐
    │ 优先 │ 计划 │
    │ 执行 │ 执行 │
 低 ├─────┼─────┤ 高
 努 │ 考虑 │ 避免 │
 力 │ 委托 │ 执行 │
    └─────┼─────┘
          │
        低影响
```

**使用方法**：
1. 列出所有待办事项
2. 评估每个事项的「影响」（对目标的贡献度）和「努力」（所需时间/资源）
3. 放入矩阵对应象限
4. 执行顺序：优先执行 > 计划执行 > 考虑委托 > 避免执行

---

## 5. 风险矩阵

```
        高影响
          │
    ┌─────┼─────┐
    │ 高   │ 极高 │
    │ 优先 │ 优先 │
 低 ├─────┼─────┤ 高
 概 │ 中   │ 高   │
 率 │ 优先 │ 优先 │
    └─────┼─────┘
          │
        低影响
```

**使用方法**：
1. 识别所有潜在风险
2. 评估每个风险的「发生概率」和「影响程度」
3. 放入矩阵对应象限
4. 对「高优先」和「极高优先」风险制定应对措施

## 6. 5-Why 分析

```
问题：[表面现象]
  ↓ 为什么？
原因 1：[直接原因]
  ↓ 为什么？
原因 2：[深层原因]
  ↓ 为什么？
原因 3：[更深层原因]
  ↓ 为什么？
原因 4：[系统性原因]
  ↓ 为什么？
根因 5：[根本原因]
```

**使用方法**：
1. 陈述表面问题
2. 连续追问 5 次"为什么"（或直到无法继续追问）
3. 最后一个"为什么"的答案就是根因
4. 针对根因制定解决方案（而非针对表面现象）

**注意**：每次追问的方向可能不同，建议对同一问题做 2-3 条 5-Why 链，交叉验证。

---

## 7. 第一性原理思考

**核心方法**：将问题分解到最基本的真理，然后从零开始重新构建解决方案。

**步骤**：
1. **识别假设**：列出当前方案中所有"理所当然"的假设
2. **质疑假设**：每个假设真的是必须的吗？有没有例外？
3. **分解到基本元素**：这个问题的最基本组成部分是什么？
4. **重新构建**：从基本元素出发，有没有全新的组合方式？

**适用场景**：
- 现有方案都不满意时
- 行业惯例可能不再适用时
- 需要突破性创新时
