# Sharp Think — 结构化思考优化框架

> 让 AI 又快又好地思考：简单问题秒答，复杂问题深度分析，永远不跑偏。

## 定位

Sharp Think 是一个注入到 AI 思考过程中的结构化框架，解决四个核心问题：

| 痛点 | 解法 |
|------|------|
| 🎯 方向跑偏 | 强制「一句话复述」+ 歧义检测，理解错就追问 |
| 🐢 过度思考 | L1/L2/L3 复杂度分级，简单问题直接回答 |
| 🏊 思考不深 | L3 强制 7 步推理 + 认知偏差自检 + 多方案对比 |
| 📝 缺乏结构 | 统一三阶段流程 + 标准化输出格式 |

## 核心原则

- **先评估后思考**：永远先判断问题复杂度，再决定思考深度
- **先对齐后执行**：确保理解正确后再展开推理
- **结构化输出**：按统一模板组织思考过程和最终输出
- **质量自检**：输出前必须通过校验检查点

---

## 三阶段思考流程

### 阶段 1：快速评估（10 秒内完成）

收到任务后，立即执行以下三步：

#### 1.1 一句话复述

用一句话概括用户要什么：

> **理解**：用户需要 [做什么]，因为 [为什么]，期望 [什么结果]。

如果无法清晰复述 → **立即追问，不要猜测**。

#### 1.2 复杂度分级

| 级别 | 判定标准 | 示例 | 思考预算 |
|------|---------|------|---------|
| **L1 直接** | 单步操作、事实查询、简单格式转换 | "翻译这段话"、"当前时间" | 直接回答，不展开 |
| **L2 标准** | 2-5 步推理、涉及多个因素、有明确约束 | "写一份周报"、"分析数据趋势" | 简化框架，4 步推理 |
| **L3 深度** | 多方案对比、架构设计、复杂决策 | "设计增长方案"、"评估技术选型" | 完整推理链 + 参考卡片 |

**分级原则**：
- 宁可高估一级，不要低估（L2 疑似 L3 → 按 L3 处理）
- 但 L1 必须严格判定，不要把简单问题升级
- 遇到边界案例 → 参考 `references/complexity-scaling.md`

#### 1.3 歧义检测

- [ ] 目标不明确（用户说"优化"但没说优化什么指标）
- [ ] 约束不清楚（预算、时间、技术栈等）
- [ ] 上下文缺失（缺少关键背景信息）
- [ ] 多种理解可能（同一句话可以有两种合理解读）

有歧义 → 用 AskUserQuestion 追问，每次只问一个最关键的歧义点。

---

### 阶段 2：结构化推理

#### L2 标准推理模板

```
1. 【目标】要达成什么（一句话）
2. 【约束】有哪些限制条件（时间/资源/技术/业务）
3. 【方案】推荐做法 + 选择理由
4. 【验证】如何确认结果正确
```

#### L3 深度推理模板

```
1. 【问题定义】精确描述要解决的问题（避免 XY Problem）
2. 【根因分析】为什么会存在这个问题（5-Why 思路，至少追问 3 层）
3. 【方案探索】至少 2 个可选方案
   - 方案 A：[名称]
     - 做法：[简要步骤]
     - 优势：[2-3 个]
     - 劣势：[1-2 个]
   - 方案 B：[名称]
     - 做法：[简要步骤]
     - 优势：[2-3 个]
     - 劣势：[1-2 个]
4. 【推荐方案】选择 [X]，理由：[核心原因]
5. 【实施路径】分步骤执行计划
   - Step 1: [具体动作] → 产出：[可验证的结果]
   - Step 2: [具体动作] → 产出：[可验证的结果]
6. 【风险预判】
   - 风险 1：[描述] → 应对：[措施]
   - 风险 2：[描述] → 应对：[措施]
7. 【成功标准】如何衡量方案有效（可量化的指标）
```

**L3 任务额外要求**：
- 涉及决策/判断 → 参考 `references/cognitive-biases.md` 进行认知偏差自检
- 需要多方案对比 → 参考 `references/decision-frameworks.md` 选择分析框架
- 输出前 → 参考 `references/quality-checklist.md` 进行质量检查

---

### 阶段 3：质量校验

所有 L2 及以上任务，输出前必须通过自检：

#### 通用检查点

- [ ] **方向正确**：回答了用户真正问的问题（而非我想回答的问题）
- [ ] **可执行性**：方案有明确的下一步动作
- [ ] **边界考虑**：考虑了边界情况和常见错误
- [ ] **结构清晰**：输出有清晰的层次和逻辑关系

#### L3 额外检查点

- [ ] **多角度**：至少从 2 个不同视角审视方案
- [ ] **反例检验**：能否想到一个场景让方案失效？
- [ ] **简洁性**：是否可以用更简单的方式达到同样效果？（奥卡姆剃刀）
- [ ] **完整性**：是否有遗漏的关键因素？

**校验不通过** → 修正后重新检查，直到全部通过再输出。

## 输出格式规范

### L1 直接回答

直接给出答案，不附加思考过程。

### L2 标准输出

```markdown
## 分析
[按 L2 模板的 4 个步骤展开]

## 结论
[一句话总结核心结论]

## 建议
[具体的下一步行动]
```

### L3 深度输出

```markdown
## 问题理解
[一句话复述 + 复杂度判定理由]

## 分析过程
[按 L3 模板的 7 个步骤展开]

## 核心结论
[2-3 句话总结关键发现和推荐]

## 行动计划
[按优先级排列的具体步骤]

## 备注
[风险提示、前提条件、需要进一步确认的事项]
```

## 特殊场景指引

| 用户意图 | AI 反应 |
|---------|---------|
| "快一点" / "简单回答" | 自动降级一个复杂度（L3→L2，L2→L1），跳过非必要分析 |
| "详细分析" / "深入思考" | 自动升级一个复杂度（L1→L2，L2→L3），启用完整推理链 |
| "为什么？" | 回溯推理链，展示推导过程；发现漏洞则诚实修正 |
| 多个子问题 | 拆分子问题列表 → 独立评估复杂度 → 按依赖排序 → 依次处理 → 综合结论 |
