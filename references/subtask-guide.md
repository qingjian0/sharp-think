# 子任务管理指南

> 用于将复杂任务拆分为多个可管理的子任务，明确依赖关系和执行顺序。

---

## 何时使用子任务拆分

| 任务特征 | L3 | L4 | L5 |
|---------|----|----|----|
| 单一简单任务 | 不需要 | 不需要 | 不需要 |
| 可以清晰拆分为 2-3 个独立部分 | 可选 | 推荐 | 必须 |
| 不同部分需要不同的思考方式 | 推荐 | 必须 | 必须 |
| 有明确的先后依赖关系 | 推荐 | 必须 | 必须 |
| 预计需要多轮交互完成 | 推荐 | 必须 | 必须 |

---

## 子任务拆分原则

### 1. 单一职责
每个子任务只负责一个明确的目标，职责单一。

### 2. 可独立完成
每个子任务可以独立完成，不依赖其他子任务的中间结果（除非是明确的前置依赖）。

### 3. 可验证
每个子任务有明确的完成标准，可以验证是否完成。

### 4. 合理大小
每个子任务大小适中，过大则继续拆分，过小则合并。

## 子任务拆分方法

### 方法 A：按功能模块拆分
```
主任务：设计一个电商网站
├─ 子任务 1：设计用户登录模块
├─ 子任务 2：设计商品展示模块
├─ 子任务 3：设计购物车模块
└─ 子任务 4：设计支付模块
```

### 方法 B：按时间阶段拆分
```
主任务：产品上线
├─ 子任务 1：MVP 阶段（核心功能）
├─ 子任务 2：公测阶段（完善功能）
└─ 子任务 3：正式上线阶段（全面推广）
```

### 方法 C：按工作流程拆分
```
主任务：问题诊断与修复
├─ 子任务 1：调查研究，收集信息
├─ 子任务 2：根因分析，确定问题
├─ 子任务 3：方案设计，验证修复
└─ 子任务 4：总结复盘，文档沉淀
```

## 子任务执行流程

### Step 1：识别与定义
- 列出所有可能的子任务
- 为每个子任务定义明确的目标
- 确定每个子任务的负责人或处理方式

### Step 2：依赖关系分析
- 确定子任务之间的依赖关系
- 哪些任务必须先完成
- 哪些任务可以并行执行
- 画出依赖关系图（可选）

### Step 3：优先级排序
- 确定子任务的执行顺序
- 按依赖关系排序
- 按重要紧急排序
- 确定里程碑

### Step 4：执行与监控
- 按顺序执行子任务
- 每个子任务完成后验证结果
- 传递输出给下一个子任务
- 需要调整时重新规划

### Step 5：总结与汇总
- 汇总所有子任务的结果
- 形成最终输出
- 总结经验教训

```
1. 子任务 1 → 2. 子任务 2 → 3. 子任务 3

## 数据传递规则

每个子任务的输出必须显式传递给下一个子任务。
- 子任务 1 输出 → 传递给子任务 2 作为输入
- 子任务 2 输出 → 传递给子任务 3 作为输入
```

## 子任务检查清单

- [ ] 拆分完整：所有子任务加起来覆盖原任务
- [ ] 粒度适中：每个子任务可以在 1 个会话内完成
- [ ] 依赖清晰：明确子任务之间的依赖关系
- [ ] 独立可验证：每个子任务有可验证的完成标准
- [ ] 结果可汇总：子任务结果可以合并为最终答案
- [ ] 数据量合理：每个子任务的数据量适中，避免一次性处理过多数据

## 数据处理优化策略

### 1. 数据分批次处理
- **做法**：将大任务的数据拆分为多个批次，每次处理一个批次
- **优势**：减少单次处理的数据量，降低系统负载
- **适用场景**：数据量较大且可以按批次处理的任务

### 2. 子任务拆分粒度优化
- **做法**：调整子任务的拆分粒度，将大子任务拆分为更小的子任务
- **优势**：每个子任务数据量小，处理速度快，不易堵塞
- **适用场景**：任务逻辑清晰，可以进一步细分子任务的场景

### 3. 数据压缩和摘要
- **做法**：对处理的数据进行压缩或生成摘要，减少写入的内容量
- **优势**：直接减少数据量，保持核心信息
- **适用场景**：数据中有大量冗余信息的场景

### 4. 异步处理机制
- **做法**：将子任务处理改为异步模式，不阻塞主流程
- **优势**：不影响主流程执行，提高整体效率
- **适用场景**：子任务处理时间较长的场景

## 常见问题处理

### Q：子任务之间有冲突怎么办？
A：重新分析问题，调整子任务划分，确保不冲突。

### Q：执行过程中发现子任务拆分不合理？
A：暂停执行，重新规划子任务拆分，不要强行推进。

### Q：某个子任务比预期更复杂？
A：可以将该子任务继续拆分，或者调整其他子任务的范围。

### Q：子任务处理数据过多导致堵塞？
A：采用数据分批次处理、优化子任务拆分粒度、数据压缩和摘要等策略，减少单次处理的数据量。
