# L1-L5 复杂度决策矩阵

> 用于精准判断任务复杂度，选择合适的分析框架。

---

## 维度定义

| 维度 | 说明 |
|------|------|
| 推理步数 | 需要多少步逻辑推导才能得出结论 |
| 方案对比 | 是否需要对比多个可选方案 |
| 决策影响 | 决策的影响范围和不可逆程度 |
| 涉及人数 | 影响的利益相关方数量 |
| 时间跨度 | 任务的时间周期跨度 |

---

## L1-L5 决策矩阵

| 维度 | L1 | L2 | L3 | L4 | L5 |
|------|-----|-----|-----|-----|-----|
| 推理步数 | 0（直接回答） | 2-3 步 | 3-5 步 | 5+ 步 | 10+ 步 |
| 方案对比 | 不需要 | 可选（1-2 个） | 可选（2 个） | 必须（2+ 个） | 必须（3+ 个） |
| 决策影响 | 无 | 低 | 中 | 高 | 重大 |
| 涉及人数 | 个人 | 小团队 | 团队 | 部门 | 公司/多部门 |
| 时间跨度 | 即时 | 天 | 周 | 月 | 季度/年 |
| 子任务拆分 | 不需要 | 不需要 | 可选 | 推荐 | 必须 |
| 参考卡片引用 | 无 | 可选 | 质量检查 | 决策框架 | 全部卡片 |

---

## 分级判定规则

### L1 直接回答
**满足任意一条即可判定为 L1**：
- 事实查询（当前时间、日期、定义等）
- 单步操作（翻译、格式转换、简单计算）
- 用户明确说"快一点"或"简单回答"
- 问题有唯一正确答案

---

### L2 简要分析
**满足任意两条即可判定为 L2**：
- 2-3 步推理可完成
- 涉及简单判断
- 有明确约束但不需要多方案对比

---

### L3 标准分析
**满足任意两条即可判定为 L3**：
- 3-5 步推理可完成
- 涉及多个因素考虑
- 可选子任务拆分

---

### L4 深度分析
**满足任意两条即可判定为 L4**：
- 5+ 步推理完成
- 需要多方案对比（至少 2 个）
- 决策影响中等偏高
- 推荐子任务拆分

---

### L5 全面研究
**满足任意两条即可判定为 L5**：
- 复杂决策，影响重大
- 需要多维度分析
- 涉及多个利益相关方
- 必须子任务拆分和分阶段验证

---

## 易混淆场景说明

| 用户说的 | 实际等级 | 判断理由 |
|---------|---------|---------|
| "简单分析一下" | 看内容 | 看问题复杂度，不听关键词 |
| "帮我做个决定" | L4/L5 | 涉及决策，需要多方案对比 |
| "给个建议" | 看影响 | 影响大→L4/L5，影响小→L2/L3 |
| "快速回答" | 可以降级 | 用户明确要求简单化 |
| "全面一点" | 可以升级 | 用户明确要求深入分析 |

---

## 动态调整规则

| 用户指令 | 调整方式 |
|---------|---------|
| "快一点"/"简单回答" | 降级 1-2 级复杂度 |
| "详细分析"/"深入思考" | 升级 1-2 级复杂度 |
| "我只要结论" | 降级到 L1/L2，只给结论 |
| "还有其他方案吗？" | 升级到 L4+，增加方案探索 |
| "分步来"/"拆成子任务" | 升级 1 级 + 启用子任务拆分 |

---

## 边界案例处理

| 情况 | 建议 |
|------|------|
| 不确定是 L1 还是 L2 | 按 L2 处理（多花 10 秒比漏掉重要因素好） |
| 不确定是 L2 还是 L3 | 按 L3 处理（宁可多分析一步） |
| 不确定是 L3 还是 L4 | 按 L4 处理（更稳妥） |
| 用户似乎问了一个简单问题，但直觉告诉你背后有更复杂的需求 | 先按用户字面意思回答（L1/L2），然后追问"是否需要更深入的分析？" |
| 任务可以拆分为多个独立子问题 | 对每个子问题独立分级，但整体框架按最高级别处理 |

---

## 快速判断流程图

```
收到任务
    ↓
一句话复述确认意图 → 歧义则追问
    ↓
判断复杂度
    ├─ 事实查询/单步操作 → L1
    ├─ 2-3 步推理 → L2
    ├─ 3-5 步推理 → L3
    ├─ 5+ 步推理/多方案对比 → L4
    └─ 复杂决策/多维度 → L5
    ↓
调用对应子技能
```
