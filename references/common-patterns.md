# 常见模式库

> 收集常见任务的标准化处理流程，遇到类似场景可以直接套用。

---

## 模式 1：技术选型决策

**适用场景**：
- 选择技术栈
- 选择框架/库
- 选择架构方案

**标准流程**：
1. 调查研究：收集候选方案（至少 3 个）
2. 确定评估维度：性能、开发成本、学习曲线、生态、维护等
3. 决策矩阵：用决策矩阵打分
4. 风险评估：每个方案的潜在风险
5. 推荐方案：给出选择和理由
6. 实施路径：分阶段切换建议

**推荐复杂度**：L4-L5

**参考工具**：决策矩阵、风险评估、调查研究

---

## 模式 2：问题诊断与修复

**适用场景**：
- Bug 根因分析
- 性能问题诊断
- 异常现象排查

**标准流程**：
1. 调查研究：收集现象、复现条件、已排除原因
2. 矛盾分析：识别主要矛盾，提出假设
3. 集中兵力：验证假设，逐一排查
4. 实践认知：如果假设被证伪，回到第 2 步
5. 批评复盘：总结经验，防止下次犯同样错误

**推荐复杂度**：L3-L4

**参考工具**：5-Why、矛盾分析、调查研究

---

## 模式 3：方案设计与迭代

**适用场景**：
- 功能设计
- 流程设计
- 架构设计

**标准流程**：
1. 调查研究：理解需求、约束、现有条件
2. 矛盾分析：识别核心问题
3. 方案探索：至少 2 个方案
4. 推荐方案：选择最优
5. 实施路径：分阶段执行
6. 迭代优化：收集反馈，持续改进

**推荐复杂度**：L3-L4

**参考工具**：子任务拆分、决策框架

---

## 模式 4：规划执行与里程碑

**适用场景**：
- 项目规划
- Roadmap 制定
- 分阶段实施

**标准流程**：
1. 调查研究：理解目标、资源、约束
2. 矛盾分析：识别关键问题和依赖
3. 子任务拆分：拆分为可管理的任务
4. 集中兵力：确定优先级和里程碑
5. 持久战略：分阶段执行，逐步推进

**推荐复杂度**：L4-L5

**参考工具**：子任务管理、持久战略、集中兵力

---

## 模式 5：调查研究与信息收集

**适用场景**：
- 新项目/新领域入门
- 决策前信息收集
- 问题诊断前期

**标准清单**：
- [ ] 现状是什么？（事实层面）
- [ ] 有哪些约束条件？
- [ ] 历史上是怎么做的？
- [ ] 别人是怎么做的？
- [ ] 用户/利益相关方有什么反馈？
- [ ] 有哪些已知的未知？
- [ ] 有哪些可依托的基础？

**信息来源**：
- 文档和文件（项目文档、代码库）
- 历史记录（历史对话、历史决策）
- 相关方沟通（用户、团队、利益相关者）
- 最佳实践（行业标准、同类项目）
- 实测结果（测试、实验、数据）

**推荐复杂度**：L2-L4

**参考工具**：调查研究

---

## 模式选择指南

| 你的情况 | 推荐模式 |
|---------|---------|
| 选择技术栈/框架/方案 | 模式 1：技术选型决策 |
| Bug/性能问题诊断 | 模式 2：问题诊断与修复 |
| 功能/流程/架构设计 | 模式 3：方案设计与迭代 |
| 项目规划/Roadmap | 模式 4：规划执行与里程碑 |
| 新项目/新领域入门 | 模式 5：调查研究与信息收集 |
| 以上都不完全符合 | 参考工作流组合，或单独调用工具 |
